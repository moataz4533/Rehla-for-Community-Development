# دليل إعداد لوحة التحكم — رحلة للتنمية المجتمعية

لوحة التحكم متاحة على المسار `/admin`، محمية بالكامل: لا يدخلها إلا
مستخدم (1) مسجّل في نظام مصادقة Supabase، و(2) موجود في جدول
`admin_users` بحالة نشطة.

## الخطوة 1: تفعيل نظام المصادقة في Supabase

1. في Supabase Dashboard → **Authentication** → **Providers**
2. تأكد أن **Email** مفعّل
3. (اختياري لكن مُستحسن للبداية) في **Authentication → Providers → Email**،
   يمكنك إيقاف "Confirm email" مؤقتًا لتسهيل إنشاء أول حساب، ثم إعادة
   تفعيله لاحقًا.

## الخطوة 2: إنشاء أول مستخدم (المدير العام)

1. في Supabase Dashboard → **Authentication** → **Users** → **Add user**
2. أدخل البريد الإلكتروني وكلمة سر قوية للمدير
3. اضغط **Create user**
4. **انسخ الـ User UID** (معرّف المستخدم الذي ظهر) — ستحتاجه في الخطوة التالية

## الخطوة 3: ربط المستخدم بجدول admin_users

افتح **SQL Editor** في Supabase ونفّذ (مع استبدال القيم):

```sql
insert into admin_users (auth_user_id, full_name, email, role, is_active)
values (
  'الصق-الـ-User-UID-هنا',
  'اسم المدير الكامل',
  'بريد-المدير@example.com',
  'super_admin',
  true
);
```

## الخطوة 4: تسجيل الدخول

1. افتح `https://rehla-foundation.vercel.app/admin-login`
2. أدخل البريد وكلمة السر
3. ستنتقل تلقائيًا للوحة التحكم

## إضافة مستخدمين إداريين آخرين لاحقًا

كرّر الخطوتين 2 و3، مع اختيار الدور المناسب في الخطوة 3:
- `super_admin` — صلاحيات كاملة (إدارة كل شيء + المستخدمين)
- `content_editor` — إدارة المحاور والحالات والأخبار
- `finance_viewer` — عرض التبرعات والتقارير فقط

## ملاحظة أمنية مهمة

متغير البيئة `SUPABASE_SERVICE_ROLE_KEY` ضروري لعمل لوحة التحكم وAPI
التبرعات. تأكد من إضافته في إعدادات Vercel، وأنه **لا يظهر أبدًا** في أي
ملف مرفوع على GitHub.
